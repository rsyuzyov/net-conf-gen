Избегать англицизмов вроде «флакс/flaky», «merge» (когда можно «слить»/«объединить»), «replace» (когда можно «заменить»), «preserved» (когда можно «сохраняется»). Технические термины из кода/инструментов (`scan_status`, `commit`, `force`, `cron`, имена методов) — можно как есть.

**Почему:** пользователь сделал замечание 2026-05-15 — англицизмы засоряют русский язык, читать их неприятно. «Используй устоявшиеся общепринятые слова, не калькуй с английского без нужды.»

**Как применять:** перед отправкой ответа просмотреть на наличие калек с английского, заменить на русские эквиваленты. «flaky сеть» → «нестабильная сеть», «merge» → «слить»/«объединить», «replace» → «заменить», «preserve» → «сохранить»/«не трогать», «edge case» → «крайний случай». Технические термины, у которых нет нормального русского аналога (TCP, SSH, JSON, API), оставлять как есть.
